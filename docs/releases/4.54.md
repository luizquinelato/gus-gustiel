## 📣 What's New

This release introduces two new documentation exports — **Release Notes** and **Ideas Backlog** — and reshapes how Gustiel documents itself, plus two user-visible bug fixes carried over from the 4.53 patch series. This is the **final v4 release**; the next consolidation will ship as `v5.0`.

### 📣 New action — `export-release-notes`

Gustiel now publishes a dedicated **Release Notes** page tree to Confluence, separate from the User Guide and Architecture Guide.

The page tree is a three-tier hierarchy with **stable titles** so re-runs replace the tree in place. The root and `vX` pages are **organizers** (no body content) — only the per-release pages carry the actual notes:

```
📣 Gustiel Release Notes        ← Root page (organizer)
  ├── v5                         ← Major bucket (organizer, newest first)
  │     ├── v5.1 — …
  │     └── v5.0 — …
  └── v4
        ├── v4.55 — …
        └── v4.54 — Release Notes & Documentation Lifecycle
```

The export is **additive only** — deleting a release file from `docs/releases/` and re-exporting will not remove the corresponding Confluence page; it stays orphaned under its `vX` bucket. Manual cleanup in Confluence is required if you actually want a release page gone.

Trigger phrases: *"export release notes"*, *"export the changelog"*, *"update the release notes page"*.

### 💡 New action — `export-ideas-backlog`

Gustiel now publishes a forward-looking **Ideas Backlog** to Confluence — a place to share proposals and explorations open for feedback. Nothing in the backlog is committed; it's the counterpart to the Release Notes (which capture what already shipped).

The page tree is two tiers with **stable titles**:

```
💡 Gustiel Ideas Backlog        ← Root page (organizer, short intro)
  ├── <Idea A>                   ← per-idea pages, alphabetical
  ├── <Idea B>
  └── <Idea C>
```

Unlike the additive Release Notes, this export uses **destructive sync** scoped strictly to the immediate children of the backlog root: any child page whose title doesn't match a current idea file in `docs/ideas/` is **deleted**. Renaming or deleting an idea file removes the corresponding Confluence page on the next export. ⚠️ Confluence comments live on the page, so deleting an idea also deletes its comments — feedback should be shared via chat or talk, not Confluence comments.

Trigger phrases: *"export ideas backlog"*, *"export Gustiel ideas"*, *"export the backlog"*.

### 🧭 New default for "export docs"

When the user asks for *"export docs"*, *"export everything"*, *"export all"*, or *"export both"*, Gustiel now exports **all four** documentation pages in one go: User Guide, Architecture Guide, Release Notes, and Ideas Backlog. Single-page exports remain available by name.

### 🏷️ Stable documentation titles

The User Guide and Architecture Guide pages no longer carry a date or author suffix in their Confluence title. They now use the stable titles **`📖 Gustiel User Guide`** and **`🏗️ Gustiel Architecture Guide`** so re-running the export updates the existing page in place instead of creating a new dated copy each time.

Portfolio and team reports are unchanged — they keep their `[YYYY-MM-DD] … by [email]` titles because they are point-in-time snapshots.

### 🏗️ Renamed: Technical Reference → Architecture Guide

The developer-focused export is now consistently called the **Architecture Guide** in the page title, success messages, and the prompt's documentation routing.

### 🐛 Fixed — Confluence image previews

Screenshots embedded in the User Guide page were uploading successfully but Confluence rendered them as "Preview unavailable". Adding the missing **`write:confluence-file`** scope and re-deploying restores image previews on every export. (Carried over from the 4.53 patch series.)

### 🐛 Fixed — `list admins` showing only account IDs

The `list admins` command was returning bare account IDs without name/email enrichment. The user-profile lookup is restored so admins now appear with their display name and email. (Carried over from the 4.53 patch series.)

## 🔧 Technical Changes

### Documentation lifecycle

- Documentation is now split into **Baseline** (User Guide + Architecture Guide — always reflects current state) and **Release Notes** (`docs/releases/MAJOR.MINOR.md` — historical delta per MINOR/MAJOR).
- Release files use front-matter (`version`, `date`, `title`) plus `## 📣 What's New` and optional `## 🔧 Technical Changes` sections.
- Release files are named `MAJOR.MINOR.md` only — PATCH bumps never produce a new file and never edit an existing one.
- New scratchpad `docs/pre-release/NEXT.md` (single optional free-form file) holds context worth preserving across a PATCH cycle. It is **not** bundled, **not** exported, and never shown to users.

### New module — `release-notes-resolver.js`

- Handles the full three-tier upsert (root → major-bucket → per-release page) in one resolver call. Root and major-bucket pages are organizers with no per-release listing in their body — Confluence's native page tree handles navigation, which keeps the export deletion-safe (orphans are intentional, never silently pruned).
- All upserts go through a shared `upsertPage` helper that follows the standard `find at target → find elsewhere (move) → create stub + update` pattern already used by the User Guide and Architecture Guide resolvers.
- Releases are grouped by MAJOR and emitted newest-first (`major desc, minor desc`) to match the bundler order.

### New module — `ideas-backlog-resolver.js`

- Two-tier upsert (root → per-idea pages) in one resolver call. Root is an organizer with a one-paragraph intro; only the per-idea pages carry full content.
- **Destructive sync** scoped strictly to the immediate children of the `💡 Gustiel Ideas Backlog` root: lists existing children → deletes any whose title isn't in the current `IDEAS` set → upserts each idea page in alphabetical order. No other Confluence content is ever touched.
- Per-idea pages are upserted with the same `find at target → find elsewhere (move) → create stub + update` helper used by the other doc resolvers.

### New service helpers — `confluence-api-service.js`

- `getPageChildren(pageId)` — lists the immediate page children of a parent, transparently following v2 pagination.
- `deleteConfluencePage(pageId)` — deletes a page via the v2 API; treats 404 as success for idempotency. Used **only** by the Ideas Backlog resolver.

### Bundler additions — `scripts/bundle-docs.mjs`

- Reads every `docs/releases/*.md` file, parses front-matter, extracts `📣 What's New` and `🔧 Technical Changes`, and emits a `RELEASES` array in `src/docs/index.js`.
- Reads every `docs/ideas/*.md` file, parses front-matter (`title`, `status`, `created`) plus freeform body, and emits a new `IDEAS` array sorted alphabetically by title.
- The `docs/pre-release/` folder is skipped.

### Manifest / wiring

- New actions `export-release-notes` and `export-ideas-backlog`, plus matching function entries, in `manifest.yml`.
- `exportReleaseNotes` and `exportIdeasBacklog` exported from `src/index.js`.
- `write:confluence-file` scope added (required by the v1 attachment endpoint used by `uploadAttachment`).
- `delete:page:confluence` scope added (required by `deleteConfluencePage`, used only by the Ideas Backlog resolver).
- `prompts/main_prompt.md` Skill 6 routing updated: "docs / everything / all / both" now maps to all four doc actions; release-notes-only and ideas-backlog-only triggers added.

### Resolver clean-up

- `skill-docs-resolver.js` and `architecture-guide-resolver.js` no longer compute a date+email page title. They drop the `getUserEmail` / `getCurrentAccountId` calls entirely and use stable titles. The "Created at" timestamp inside the page body is kept (it shows when the export last ran, not who ran it).
- `architecture-guide-resolver.js` page H1, success message, and module doc-comment renamed from "Technical Reference" to "Architecture Guide".
- `storage-admin-resolver.js`'s `fetchUserProfile` was rebuilt around the dual-attempt `asUser` / `asApp` pattern with proper imports — fixing the `ReferenceError` that surfaced after the 4.53 data refresh.
